> How to maintain existing PostgreSQL clusters with Pigsty?

This section provides standard operating procedures (SOP) for common PostgreSQL administration tasks:

- [SOP](sop): Standard operating procedures for creating/removing clusters and instances, backup & restore, rolling upgrades, etc.
- [Failure](failure): Common failure troubleshooting strategies and handling methods, such as disk exhaustion, connection exhaustion, XID wraparound, etc.
- [Drop](drop): Emergency procedures for handling accidental data deletion, table drops, and database drops
- [Maintain](maintain): Maintenance tasks including regular inspections, post-failover cleanup, bloat management, VACUUM FREEZE, etc.
- [Tuning](tune): Automatic optimization strategies and adjustment methods for memory, CPU, storage parameters, etc.
